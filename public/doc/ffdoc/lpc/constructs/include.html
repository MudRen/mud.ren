LPC 的 #include 命令:

语法:

    #include <file.h>
另一种语法:
    #include "file.h"
说明:
    #include "file.h" 在目前的目录中寻找 file.h.
    #include <file.h> 则在标准的系统引用目录 (standard system include directories) 中寻找 file.h (TMI mudlib 中, 此种目录是 /include 和 /local/include).
熟悉 C 的人应该知道, LPC 的 #include 语句与 C 的相同. 而对于不熟悉的人来说, #include 是引用 (include) 其他文件内容的方法. 把 #include "file.h" 放在一个文件中, 等于把 file.h 的内容全部写在您摆 #include "file.h" 的位置. 每次物件重新编译时, 物件的引用档也会重新编译一次. 如果引用档里面有变量或函数名称与物件中的一样, 编译时会产生双重命名 (duplicated-name error) 的错误 (如果您把 file.h 中的内容全部写在物件中, 结果一样).
