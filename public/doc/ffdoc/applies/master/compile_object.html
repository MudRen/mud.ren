compile_object(4) MudOS v21c2　 (5 Sep 1994)

　名称:
    　 compile_object - 提供虚拟物件 (virtual object) 功能的 mudlib　 介面
　语法:
    　 object compile_object( string pathname );

    　 物件 compile_object( 字符串 pathname );

　用法:
    　 当 mudlib 指示驱动程序载入一个不存在于内存中的文件, 驱动程序会调用 compile_object().　 举例来说, 如果 mudlib　 调用 call_other("/obj/file.r", "some_function") 或 new("/obj/file.r") 时, 驱动程序就调用主控物件 (master.c) 中的 compile_object("/obj/file.r"). 而 /obj/file.r　 为不存在于内存的文件名称. 如果 mudlib　 不希望连结 (associate)　 名为 /obj/file.r
    　 的文件, compile_object()　 将返回 0. 如果 mudlib 愿意, 就要返回 /obj/file.r　 这个物件. 在 compile_object() 连结一个文件名称和物件之后, 对驱动程序而言, 就如同 file.r.c　 这个文件存在于游戏中. 载入并产生此物件时, 就是 compile_object() 返回的物件.
